## 💼 Featured Projects

### 🌐 High-Throughput Financial Ledger Engine
*   **Business Problem:** Handling inconsistent multi-region transactional requests while guaranteeing atomic transaction safety without severe database bottlenecking.
*   **Architecture & Stack:** `Go`, `gRPC`, `Apache Kafka`, `CockroachDB`, `Redis`, `AWS EKS`.
*   **Key Engineering Engineering Challenges:** Mitigating transactional race conditions under massive concurrent write spikes. Solved by implementing an event-driven event-sourcing pattern decoupled by distributed write-ahead logging (WAL) pipelines.
*   **Production Readiness:** Achieved sub-50ms p99 execution latencies with horizontal capability up to 85,000 requests per second. Integrated strict TLS mutual authentication (mTLS) and envelope encryption for data at rest.

### 🤖 Multi-Agent Generative RAG Pipeline
*   **Business Problem:** Standard semantic enterprise searches suffer from low context precision and poor document retrieval metrics over multi-structured file systems.
*   **Architecture & Stack:** `Python`, `FastAPI`, `LangChain`, `Pinecone Vector DB`, `PostgreSQL`, `Docker`.
*   **Key Engineering Engineering Challenges:** Orchestrating dynamic synchronization between continuously shifting relational schemas and multi-dimensional vector indices. Developed an asynchronous Change Data Capture (CDC) processing worker handling token chunk distributions reliably.
*   **Production Readiness:** Fully decoupled layout utilizing strict semantic caching layer to drop LLM API consumption cost constraints by roughly 42%.
